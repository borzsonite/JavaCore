1. Отношение = таблица, атрибут = колонка, кортеж = строка, мощность = кол-во строк, размерность = кол-во колонок, домен = ячейка.
2. Имя таблицы должно начинаться с буквы
3. Ключевое поле id,  является ОБЯЗАТЕЛЬНЫМ ЭЛЕМЕНТОМ каждой реляционной таблицы.
4. Ключевое поле  должно состоять  из двух частей: начинаться с названия таблицы, которой поле принадлежит, затем через подчеркивание необходимо указать id, например book_id
5. Типы данных в SQL: int, decimal(вещественное число в скобка указывается количество сиволов слева и справа от запятой), date, varchar(строка не более 255 символов)
6. Описание ключевого поля INT PRIMARY KEY AUTO_INCREMENT
7. Алгоритм создание таблицы:
    - ключевое слово CREATE TABLE (можно маленькими буквами)
    - название таблицы
    - скобка
    - имя поля (атрибута) + его описание, которое включает тип поля и другие необязательные характеристики
    - запятая
    - имя поля
    - ....
    - закрываающая скобка
8. Запрос можно писать на нескольких строках в конце точка с запятой, если запрос один - необязательо
9. Алгоритм вставки значений в таблицу:
    - ключевые слова INSERT INTO
    - имя таблиц
    - (
    - список полей в которые будут вноситься значения
    - )
    - ключевое слово VALUES
    - (
    - значения
    -)
10. Алгоритм выбора всех записей из таблицы: SELECT * FROM table_name
11. Алгоритм выбора нескольких записей из таблицы: SELECT имя_столбца1, имя_столбца2, итд. FROM table_name
12. Алгоритм выбора отдельных столбцов и присвоение им новых имен:
    - SELECT имя_столбца AS новое_имя, имя_столбца AS новое_имя, etc.. FROM table_name
13. Алгоритм  выборки данных с созданием вычисляемого столбца:
    - SELECT имя_столбца1, имя_столбца2, итд. имя_столбца1 * имя_столбца2 AS новое_имя FROM table_name
